## 一、为什么要学习 Markdown

Markdown 的核心理念，是用少量简单符号在打字的同时完成排版。写作者不需要频繁离开键盘去调整格式，思路不容易被打断，因此可以更专注于内容本身。

![[笔记同步助手/images/d4d08bc064d4a362c03fc66d7319f9ff_MD5.jpg|视频开头介绍 Markdown 入门教程主题]]

视频开头介绍 Markdown 入门教程主题

Markdown 也具有很强的通用性。许多知识管理工具以 Markdown 为基础，例如 Obsidian、Joplin 等；Notion、思源笔记等工具虽然不完全以 Markdown 为核心，但使用 Markdown 也是非常高效的输入方式。

在博客、论坛、公众号排版等场景中，Markdown 也被广泛支持。通过 Markdown 工具，可以把纯文本快速转换为适合发布平台的排版效果。

在 AI 时代，Markdown 还是一种重要的沟通格式。AI 经常以 Markdown 输出内容，而我们编写提示词时，也可以用 Markdown 清晰地定义角色、背景、任务、输出格式等信息。

Markdown 文件通常以 .md 为扩展名，可以在不同电脑和系统中打开阅读，也可以转换为 Word、PDF、PPT 等常见格式。

## 二、学习前的注意事项

本教程主要介绍基础且通用的 Markdown 语法。有些语法虽然常见，但并不是所有编辑器都支持，例如自动生成目录的特定写法、任务列表、脚注、Obsidian 增强语法等。

![[笔记同步助手/images/617de6fd52f1751bf4b2bc09dcd47795_MD5.jpg|Markdown 语法表总览页面]]

Markdown 语法表总览页面

Markdown 中也可以嵌入 HTML 标签，例如下划线标签 <u>，但在日常写作场景中通常不需要直接书写 HTML，因此基础学习阶段可以暂时忽略。

所有 Markdown 符号都应使用英文输入法下的半角符号，例如 #、\*、-、>、|、\` 等。

## 三、标题与基础文本格式

### 1\. 标题

标题使用井号加空格表示，井号数量对应标题级别。例如一个井号是一级标题，两个井号是二级标题，最多可到六级标题。标题通常也是文章大纲和目录生成的依据。

![[笔记同步助手/images/cef0cef0b8ea90f13ab6fce58e870e50_MD5.jpg|左侧 Markdown 源码与右侧预览模式对比展示标题语法]]

左侧 Markdown 源码与右侧预览模式对比展示标题语法

示例：# 一级标题、## 二级标题、### 三级标题。

### 2\. 普通文本、粗体与斜体

普通文本直接输入即可。需要强调文字时，可以使用星号包裹文字：前后各两个星号表示粗体，前后各一个星号表示斜体，前后各三个星号表示粗体加斜体。

-   粗体：\*\*文字\*\*
-   斜体：\*文字\*
-   粗体加斜体：\*\*\*文字\*\*\*

部分 Markdown 编辑器支持使用两个等号实现高亮，例如 ==高亮文字==；删除线通常使用两个波浪线包裹，例如 ~~删除文字~~。

## 四、列表语法

### 1\. 有序列表与无序列表

列表分为有序列表和无序列表。有序列表使用数字、英文句点和空格，例如 1.；无序列表通常使用减号和空格，例如 -。输入回车后，编辑器一般会自动延续列表编号或项目符号。

![[笔记同步助手/images/265df4ac178096bdb7e785bfbd8213b3_MD5.jpg|演示有序列表和无序列表的 Markdown 写法]]

演示有序列表和无序列表的 Markdown 写法

-   有序列表：1. 空格后输入内容
-   无序列表：- 空格后输入内容
-   无序列表也可使用 + 或 \*，但通常推荐统一使用 -

如果需要嵌套列表，可以使用 Tab 键缩进；如果要取消缩进，可以使用 Shift + Tab。

### 2\. 任务列表

任务列表并非最基础的原生 Markdown 标准语法，但现在已经被许多工具广泛支持。它适合用于待办清单。

-   未完成任务：- \[ \] 任务内容
-   已完成任务：- \[x\] 任务内容

## 五、段落、引用、换行与分割线

### 1\. 引用

引用使用大于号加空格表示，例如 > 引用内容。如果需要嵌套引用，可以连续输入多个大于号，数量越多，嵌套层级越深。

![[笔记同步助手/images/fb7aa2a901a633eeba6ac7a74911a7d7_MD5.jpg|引用、强制换行和分割线语法展示]]

引用、强制换行和分割线语法展示

### 2\. 强制换行

在同一个段落内如果需要强制换行，可以在行尾输入两个空格后再回车。这样视觉上会换行，但仍保持在同一段落结构中。

### 3\. 分割线

分割线可以用三个或三个以上的减号表示，例如 ---。输入后会生成一条横向分割线，用于区分文章中的不同部分。

## 六、链接、锚点链接与脚注

### 1\. 普通链接

链接语法由方括号和小括号组成，方括号中写链接显示文字，小括号中写链接地址。

示例：\[链接文字\](https://example.com)。

### 2\. 引用式链接

当文章中多次出现同一个链接，或在起草阶段暂时不想直接写完整网址时，可以使用引用式链接。正文中先写链接标题和引用名称，再在文章末尾统一定义引用名称对应的网址。

示例：\[链接文字\]\[ref\]，文章末尾写 \[ref\]: https://example.com。

![[笔记同步助手/images/e2b64046596312701af67e19ebad00c6_MD5.jpg|链接、引用式链接和脚注语法示例]]

链接、引用式链接和脚注语法示例

### 3\. 脚注

脚注常见于百科类或学术类内容。正文中使用 \[^1\] 标记脚注位置，文章末尾用 \[^1\]: 脚注内容 补充说明。脚注名称通常使用数字。

## 七、图片与带链接的图片

### 1\. 图片

图片语法与链接类似，只是在最前面多一个感叹号。方括号中写替代文字，小括号中写图片地址。图片地址可以是网络链接，也可以是本地绝对路径或相对路径。

示例：!\[图片说明\](图片地址)。

![[笔记同步助手/images/4a4ace09a635067b7cc03a2ac4f70b65_MD5.jpg|图片语法和带链接图片语法演示]]

图片语法和带链接图片语法演示

### 2\. 带链接的图片

图片语法可以嵌套在链接语法中，形成可点击跳转的图片。方法是把完整的图片语法放进链接语法的方括号中，小括号中写点击图片后要跳转的地址。

示例：\[!\[图片说明\](图片地址)\](跳转链接)。

## 八、表格语法

表格使用竖线分隔单元格。第一行是表头，第二行是分隔行，分隔行中每个单元格至少需要三个减号。可以在分隔行中使用冒号控制对齐方式。

![[笔记同步助手/images/1605a25b991214d4851dc331f1871140_MD5.jpg|Markdown 表格语法与预览效果]]

Markdown 表格语法与预览效果

-   左对齐：:---
-   右对齐：---:
-   居中对齐：:---:

分隔行之后，就是表格的具体数据行，每一行同样用竖线分隔不同单元格。

## 九、代码块与语法高亮

### 1\. 行内代码与代码块

行内代码使用一个反引号包裹，例如 \`code\`。代码块使用上下各三个反引号包裹。反引号位于键盘数字 1 左侧，输入时同样应使用英文输入法。

![[笔记同步助手/images/89680f4e2666a811181391ef02f756da_MD5.jpg|行内代码和代码块语法展示]]

行内代码和代码块语法展示

在代码块起始的三个反引号后，可以指定语言类型，例如 python、java、markdown。指定语言后，编辑器通常会提供对应的语法高亮效果。

### 2\. Mermaid 代码

如果代码块语言指定为 Mermaid，部分工具可以把代码渲染为图形结构，例如流程图、结构图或思维导图。Mermaid 与 Markdown 类似，也是一种适合文本化表达结构的语言。

![[笔记同步助手/images/3e217a048a9277c71bde38a5901b712f_MD5.jpg|Mermaid 代码生成图形结构的效果]]

Mermaid 代码生成图形结构的效果

### 3\. 代码块嵌套

如果需要在代码块内部展示另一个代码块，外层代码块需要使用多于三个反引号，例如四个反引号；内层代码块仍然使用三个反引号。这样可以避免内部代码块提前结束外层代码块。

在 Obsidian 中记录 Dataview 代码示例时，可以把外层设为 markdown 或 text，内层再写 Dataview。这样 Dataview 代码只会作为示例展示，而不会被插件实际执行。

## 十、Markdown 与 AI 提示词

AI 输出内容时，经常使用 Markdown 格式，例如有序列表、无序列表、标题、代码块等。我们也可以要求 AI 直接以 Markdown 代码块输出内容，方便复制、下载或保存为 .md 文件。

![[笔记同步助手/images/a956309c1f2353e298d182381f4d9355_MD5.jpg|AI 输出 Markdown 格式会议纪要的示例]]

AI 输出 Markdown 格式会议纪要的示例

编写 AI 提示词时，Markdown 也很适合用来组织结构。可以用标题和列表清楚划分角色、任务、背景信息、输入数据、输出格式、约束条件等模块。

需要注意的是，如果直接把 AI 生成的 Markdown 原文复制到自媒体平台、点评平台等位置，可能会保留星号、井号等 Markdown 符号，显得非常明显。更合适的做法是使用 md2card 等 Markdown 格式转换工具，先生成有样式的内容，再进行发布或排版。

### 1\. 笔记软件

Markdown 适合用于 Obsidian、Notion、思源笔记等知识管理工具。通过标题、列表、引用、代码块等语法，可以快速整理学习笔记、项目文档和个人知识库。
