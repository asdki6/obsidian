# 🔌 新仓库快速配置指南

## 插件共享机制

所有插件已集中在：

```
C:\Users\PC\Documents\obsidian-config\plugins\
```

当前仓库的 `.obsidian\plugins\` 是一个**目录链接**，指向上述共享目录。

---

## 新仓库接入步骤

打开 PowerShell，运行一行命令即可：

```powershell
New-Item -ItemType Junction `
  -Path '新仓库路径\.obsidian\plugins' `
  -Target 'C:\Users\PC\Documents\obsidian-config\plugins'
```

示例（假设新仓库在 `D:\MyNotes`）：

```powershell
New-Item -ItemType Junction `
  -Path 'D:\MyNotes\.obsidian\plugins' `
  -Target 'C:\Users\PC\Documents\obsidian-config\plugins'
```

> ⚠️ 前提：新仓库已经有了 `.obsidian` 文件夹（用 Obsidian 打开一次就会自动生成）。如果还没有，先 `mkdir '新仓库路径\.obsidian'`。

---

## 新装插件时

以后安装新插件，Obsidian 会直接下载到链接目录 → 也就是全局共享目录 → 所有仓库自动生效。

---

## 想为某个仓库单独禁用某插件？

编辑该仓库的 `.obsidian\community-plugins.json`，把不想用的插件 ID 从数组中移除即可。插件文件仍然共享，只是不加载。
