# 新仓库配置指南

> 按此操作，新建的 Obsidian vault 自动拥有全部插件和配置，无需重装。

## 原理

```
C:\Users\PC\Documents\
├── obsidian-config\          ← 共享配置目录（所有 vault 共用）
│   └── plugins\              ← 7 个插件在此，一处更新，所有 vault 生效
│       ├── calendar/
│       ├── dataview/
│       ├── obsidian-excalidraw-plugin/
│       ├── obsidian-git/
│       ├── obsidian-weread-plugin/
│       ├── realclaudian/
│       └── templater/
│
├── obsidian total repositories\   ← 主 vault（此仓库）
│   └── .obsidian\plugins\         ← 实文件，git 追踪
│
└── 新仓库\                        ← 任意新 vault
    └── .obsidian\plugins\         ← 目录链接 → 指向共享目录
```

## 建新仓库时（二选一）

### 方式 A：git clone（推荐，跨设备）
```bash
git clone https://github.com/asdki6/obsidian.git 新仓库名
```
插件、配置、知识库全部就位。

### 方式 B：本地新 vault + 目录链接
```bash
# 1. 创建新 vault 文件夹
mkdir "C:\Users\PC\Documents\新仓库名"

# 2. 创建目录链接（以管理员身份运行 cmd）
mklink /J "C:\Users\PC\Documents\新仓库名\.obsidian\plugins" "C:\Users\PC\Documents\obsidian-config\plugins"

# 3. Obsidian → 打开文件夹 → 选新仓库
```

### 方式 C：新 vault + 复制配置
直接复制 `obsidian-config\` 下的 `.obsidian` 文件夹到新仓库，再选择性保留需要的插件配置。

## 插件升级

在任一 vault 中升级插件后，如果用的是**目录链接方式**，所有 vault 自动生效。如果用的是 **git 方式**（如主仓库），提交推送即可。
