## README

Originally created by John Washam as a multi-month study plan for becoming a software engineer at a major tech company.

### What is it?

This is a **multi-month study plan** for going from a non-CS background (or CS background looking to refresh) to being a software engineer at a major company like Google, Amazon, Facebook, Microsoft, etc.

### Study Plan

The plan is divided into a series of topics that should be studied in order:

#### 1. Prerequisite Knowledge
- Basic math (arithmetic, algebra)
- Basic programming in at least one language (Python, Java, C++, etc.)

#### 2. Algorithmic Complexity / Big-O / Asymptotic Analysis
- Arrays, linked lists, time complexity
- Space complexity analysis

#### 3. Data Structures
- **Arrays** - Contiguous memory, dynamic arrays
- **Linked Lists** - Singly, doubly, circular
- **Stack** - LIFO, array/stack implementation
- **Queue** - FIFO, circular buffer, priority queue
- **Hash Table** - Hash functions, collision resolution, load factor
- **Binary Search Tree** - BST properties, traversal, balancing
- **Heap** - Min/max heap, heap operations, heap sort
- **Trie** - Prefix tree, autocomplete, spell checking
- **Graph** - Directed/undirected, adjacency matrix/list, BFS, DFS
- **Union-Find** - Disjoint set, path compression, union by rank

#### 4. Algorithms
- **Sorting** - Quick sort, merge sort, heap sort, counting sort, radix sort
- **Searching** - Binary search, depth-first search, breadth-first search
- **Recursion** - Recursive problem solving, backtracking
- **Dynamic Programming** - Memoization, tabulation, common patterns
- **Greedy Algorithms** - Interval scheduling, Huffman coding
- **Divide and Conquer** - Merge sort, quick sort, binary search
- **Graph Algorithms** - Dijkstra, Bellman-Ford, Floyd-Warshall, A*, MST (Kruskal/Prim), Topological sort

#### 5. Advanced Topics
- **System Design** - Scalability, load balancing, caching, database scaling
- **Object-Oriented Design** - Design patterns, SOLID principles
- **Concurrency** - Threads, locks, semaphores, deadlock
- **Networking** - TCP/IP, HTTP, DNS, load balancers
- **Security** - Common vulnerabilities, cryptography basics

### Recommended Resources

**Books:**
- "Cracking the Coding Interview" by Gayle Laakmann McDowell
- "Introduction to Algorithms" (CLRS)
- "The Algorithm Design Manual" by Steven Skiena

**Practice Platforms:**
- LeetCode
- HackerRank
- CodeSignal
- InterviewBit

### Daily Schedule

Suggested schedule: 8-12 hours per day for 4 months (or 4-6 hours per day for 8 months)

1. **1 hour** - Review previous day's material
2. **2-3 hours** - Study new topic
3. **2-3 hours** - Practice coding problems
4. **1 hour** - System design study (later in the plan)
5. **30 min** - Review and plan next day

### The Original Author's Journey

John Washam spent 8 months studying full-time to prepare for his software engineering interviews at Amazon. He got the job (now works at AWS), and created this guide to help others follow the same path. The repo has since become one of the most popular learning resources on GitHub.
