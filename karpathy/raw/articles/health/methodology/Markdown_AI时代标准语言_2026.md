## 核心观点

### Markdown 是 AI 时代的"标准沟通语言"
Markdown 不再是程序员的专属玩具，而是 AI 时代最通用、最省心的技能。写提示词、写知识库、写会议纪要、写项目计划——全部用 Markdown。

### 为什么 AI 喜欢 Markdown
- **结构化标记**让 AI 快速解析内容逻辑：`#` 标题、`-` 列表、`**加粗**`
- 作为 **Prompt 工程的最佳载体**，结构化输入让 AI 更精准理解需求
- 支持代码块（` ``` `）、表格、LaTeX 公式、Mermaid 流程图

### 核心语法
| 语法 | 效果 | 用途 |
|------|------|------|
| `# 标题` | 一级标题 | 文章结构 |
| `## 二级` | 二级标题 | 段落分层 |
| `- 列表` | 无序列表 | 要点罗列 |
| `**加粗**` | 粗体 | 关键词强调 |
| `[文字](url)` | 链接 | 引用来源 |
| `![描述](url)` | 图片 | 插入图片 |
| ` ```代码``` ` | 代码块 | 技术内容 |
| `> 引文` | 引用块 | 引用原文 |

### 推荐工具
- **Typora** — 所见即所得 Markdown 编辑器
- **VS Code** + Markdown 插件 — 程序员首选
- **Obsidian** — 知识管理 + Markdown 原生支持
- **Notion** — 协作场景的 Markdown 变体

### Markdown 与 Prompt 工程
写 Prompt 时，用 Markdown 结构会比纯文本得到更好的 AI 响应：
```
# 角色
你是一个技术文档作者
# 任务
用简单的话解释什么是 REST API
# 要求
- 不超过 200 字
- 用比喻
- 给一个例子
```
这是结构化 Prompt 的标准格式，AI 对结构化输入的理解准确度远高于散乱文本。
