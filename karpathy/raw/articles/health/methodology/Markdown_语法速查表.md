# Markdown 语法速查表

## 基础语法

| 语法 | 效果 | 说明 |
|------|------|------|
| `# 一级标题` | <h1>一级标题</h1> | 1-6 个 # 对应 h1-h6 |
| `## 二级标题` | <h2>二级标题</h2> | 最常用：文章段落 |
| `### 三级标题` | <h3>三级标题</h3> | 子段落 |
| `**粗体**` | **粗体** | 关键词强调 |
| `*斜体*` | *斜体* | 次要强调 |
| `~~删除线~~` | ~~删除线~~ | 标记删除 |
| `> 引用` | 引用块 | 引用原文 |
| `---` | 分割线 | 三个及以上 |

## 列表

| 语法 | 效果 |
|------|------|
| `- 无序列表` | - 项目 |
| `1. 有序列表` | 1. 第一项 |
| `- [ ] 待办` | - [ ] 未完成 |
| `- [x] 完成` | - [x] 已完成 |

## 链接与图片

| 语法 | 说明 |
|------|------|
| `[文字](url)` | 外部链接 |
| `[[文件名]]` | Obsidian 内部链接 |
| `[[文件\|别名]]` | 带别名的内部链接 |
| `![[图片.png]]` | 嵌入图片 |
| `![alt](url)` | 外部图片 |

## 代码

| 语法 | 说明 |
|------|------|
| `` `行内代码` `` | 行内代码 |
| ` ```语言 ``` ` | 代码块（指定语言高亮） |

## 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容 | 内容 | 内容 |
```

对齐：
- `:---` 左对齐
- `:---:` 居中
- `---:` 右对齐

| ------------ | -------------------- |
| `[[文件名#标题]]` | 链接到页面内标题             |
| `[[文件名^块]]`  | 链接到段落块               |

| `~~删除线~~`    | 删除线                  |
| `---` 后空行    | 前面是 YAML frontmatter |

## Prompt 工程常用结构

```markdown
# 角色
你是一个技术文档作者

# 任务
用简单的话解释什么是 REST API

# 要求
- 不超过 200 字
- 用比喻
- 给一个例子
```

## AI 友好的 Markdown 原则

1. **层级清晰**：用 `#` → `##` → `###` 让 AI 理解内容结构
2. **关键词加粗**：`**重要概念**` 让 AI 知道什么是重点
3. **列表而非段落**：AI 对列表的解析准确度远高于长段落
4. **表格结构化数据**：对比、分类、清单尽量用表格
5. **内链密集**：Obsidian 的 `[[wikilink]]` 就是 AI 的关系图谱
